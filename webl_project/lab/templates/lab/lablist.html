C Programming Lab

1.Data types & Variables, I/O statements

This experiment shows the basics of C program construction with the fundamental knowledge of data types, variables and I/O (Input/Output) statements through various simple problems.

Objectives of this experiment are:

1. To understand the basic idea of programming and fundamentals of C programming.

2.Control statements structures

This experiment shows the execution of programs which use conditional statements such as if, else-if, nested else-if, switch, GoTo.

Objectives of this experiment are:

1. To learn how decision making is done while programming.

2. To learn the control statements structures.

3.Loops

This experiment shows the execution of programs which use loops to avoid repetition. This experiment covers for loops, while loops, nested for loops and do

Objectives of this experiment are:

1. To learn how to do repetitive tasks using various looping statements.

4.Expressions

This experiment demonstrates the automatic evaluation of expressions involving arithmetic and logical operators.

Objectives of this experiment are:

1. To learn about different types of operators.

2. To learn about the precedence of the operators.

3. To learn how to evaluate an expression.

5.Arrays & Pointers

This experiment demonstrates the fundamentals of array and pointer. The similarity and use cases of array and pointers are also demonstrated in this experiment.

Objectives of this experiment are:

1. To learn how to use arrays for storing large amounts of data.

2. To understand the concept of memory address associated with every variable.

3. To understand pointer referencing and dereferencing.

4. To understand how pointers are useful in dynamic allocation of memory.

6.Function & Recursion

This experiment includes the fundamentals of functions and their calling procedure. Recursion is also included which is the process of repeating items in a self-similar way.

Objectives of this experiment are:

1. To understand that a big program can be broken up into independent modules

2. To solve such problems using recursive procedures.

3. To learn to define functions and call them with appropriate parameters.
